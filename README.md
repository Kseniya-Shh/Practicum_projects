# Practicum_projects
Проект 1. 
Изучение и анализ вариантов использования приложения, а так же анализ и зучение Use Case. Составление диаграмм в нотациях Чена и Crow's Foot, а так же нормализация диаграммы. 

Проект 2.
Работа над спецификацией требований. Необходимо обновить спецификацию, добавить функциональные требования для новых функций приложения, описать требования к ним, а так же обновить диаграмму UML Use Case; добавить новые табличные варианты использования. Дополнительно, необходимо заполнить другие разделы спецификации, а именно атрибуты качества, бизнес-правила и глоссарий.

Проект 3. 
Данный проект отражает реальные рабочие процессы в Agile-командах. 
Необходимо изучить интервью с заинтересованными лицами и проанализировать его. Далее, составить карту пользовательских историй и выделить на ней MVP. После, необходимо реализовать прототип с использованием Figma. И последняя задача проекта — составить контекстную и логическую диаграммы потоков данных (DFD).
