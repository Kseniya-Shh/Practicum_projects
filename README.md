# Practicum_projects
Проект 1. 
Задача: изучение и анализ вариантов использования приложения, а так же анализ и зучение Use Case. Составление диаграмм в нотациях Чена и Crow's Foots, нормализация диаграмм. 

Проект 2.
Задача: работа над спецификацией требований. Необходимо обновить спецификацию, добавить функциональные требования для новых функций приложения, описать требования к ним, а так же обновить диаграмму UML Use Case; добавить новые табличные варианты использования. Дополнительно, необходимо заполнить другие разделы спецификации, а именно атрибуты качества, бизнес-правила и глоссарий.

Проект 3. 
Данный проект отражает реальные рабочие процессы в Agile-командах. 
Необходимо изучить интервью с заинтересованными лицами и проанализировать его. Далее, составить карту пользовательских историй и выделить на ней MVP. После, необходимо реализовать прототип с использованием Figma. И последняя задача проекта — составить контекстную и логическую диаграммы потоков данных (DFD). 

Проект 4. 
Задача: необходимо изучить архитектуру и существующие API приложения и поставить задачи на доработку API с учетом требований заказчика. 
Необходимо изучить интервью с представителем компании, понять требования заказчика, далее нужно создать диаграмму последовательности для реализации новой функции сервиса, затем протестировать существующее API в Swagger и поставить задачу на разработку новой операции API. 

Проект 5.
Самый обширный и интересный проект. 
Необходимо изучить интервью с заказчиком и пользовтаелями, на основе чего составить карту пользовательских историй, выделить MVP, составить необходимые диаграммы (смоделировать процессы, используя диаграмму потоков данных (DFD) и смоделировать данные, используя логическую ER-диаграмму в нотации Crow’s foots), создать словарь данных и далее создать интерактивный прототип в Figma, разработать программу и методику испытаний (ПМИ).
